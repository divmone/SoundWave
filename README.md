## 📖 О проекте

SoundWave — платформа для покупки и продажи аудио-ассетов: звуковых эффектов, оповещений, переходов и джинглов. Авторы (Creator) загружают и публикуют звуки, слушатели (Listener) просматривают каталог, прослушивают превью и приобретают ассеты.

> ⚠️ Учебный прототип. Реальная оплата не реализована — факт приобретения фиксируется в БД.

## 🛠️ Стек технологий

### Backend
- **Язык:** C++17
- **HTTP-фреймворки:** Drogon, uServer
- **БД:** PostgreSQL
- **ORM:** Drogon ORM / uServer ORM
- **Авторизация:** JWT (jwt-cpp)
- **Хеширование паролей:** bcrypt (libbcrypt)

### Frontend
- **Фреймворк:** React 18 + TypeScript
- **Сборщик:** Vite
- **Маршрутизация:** React Router v6
- **HTTP-клиент:** Axios
- **Аудиоплеер:** Wavesurfer.js
- **Стилизация:** CSS Modules

## 🚀 Запуск проекта

### Требования
- PostgreSQL 14+
- Node.js 18+
- CMake 3.20+
- C++17-совместимый компилятор (GCC 11+ / Clang 14+)

### 1. Клонирование репозитория

```bash
git clone https://github.com/your-org/soundwave.git
cd soundwave
```

### 2. Настройка базы данных

```bash
psql -U postgres -c "CREATE DATABASE soundwave;"
psql -U postgres -d soundwave -f db/schema.sql
```

### 3. Запуск Backend

```bash
cd backend
mkdir build && cd build
cmake ..
make -j$(nproc)
./soundwave
```

Сервер запустится на `http://localhost:8080`.

### 4. Запуск Frontend

```bash
cd frontend
npm install
npm run dev
```

Приложение откроется на `http://localhost:5173`.

## 🔌 REST API

Базовый путь: `/api`  
Аутентификация: `Authorization: Bearer <token>`

| Метод | Эндпоинт | Описание | Доступ |
|---|---|---|---|
| POST | `/auth/register` | Регистрация | Все |
| POST | `/auth/login` | Вход, возвращает токен | Все |
| GET | `/products` | Список ассетов (фильтры, поиск, сортировка) | Все |
| GET | `/products/:id` | Данные ассета | Все |
| POST | `/products` | Создание ассета | Creator |
| PUT | `/products/:id` | Обновление метаданных | Creator (владелец) |
| DELETE | `/products/:id` | Удаление ассета | Creator (владелец) |
| PATCH | `/products/:id/status` | Изменение статуса | Creator (владелец) |
| GET | `/products/:id/file` | Скачивание файла | Авторизованный (есть в библиотеке) |
| GET | `/cart` | Содержимое корзины | Авторизованный |
| POST | `/cart` | Добавить в корзину | Авторизованный |
| DELETE | `/cart/:productId` | Удалить из корзины | Авторизованный |
| POST | `/checkout` | Оформить заказ | Авторизованный |
| GET | `/library` | Моя библиотека | Авторизованный |
| GET | `/reviews/:productId` | Отзывы на ассет | Все |
| POST | `/reviews/:productId` | Создать отзыв | Авторизованный (есть в библиотеке) |
| GET | `/profile` | Данные профиля | Авторизованный |
| PUT | `/profile` | Обновить профиль | Авторизованный |

## 🔐 Безопасность

- Пароли хранятся в виде bcrypt-хеша (10+ раундов)
- Все защищённые эндпоинты проверяют JWT-токен (HTTP 401 при отсутствии)
- Операции над ресурсами проверяют владельца (HTTP 403 при нарушении)
- Скачивание файла доступно только при наличии ассета в библиотеке
- SQL-запросы выполняются через prepared statements
